## 🔄 文档同步更新

请按 iTestAgent 项目 AGENTS.md 的规格漂移规则执行：

### 第一步：确定同步范围
1. 如果用户提供了决策内容，记录决策摘要，定位涉及的文档。
2. 如果用户未提供，读取 `docs/05-planning/task-status.json` 中的 `decisions` 记录。
3. 自动扫描关联文档。

### 第二步：执行同步
1. **规格书**（涉及 AC）：定位到 `全量用户故事与验收标准规格书.md`，按决策修改。
2. **架构设计文档**（涉及架构）：定位到对应章节，修改契约描述。
3. **技术选型文档**（涉及选型）：更新对应层级选型。
4. **数据流全链路**（涉及数据契约）：更新对应阶段的数据流。
5. **避坑手册**（涉及陷阱）：新增陷阱或标记过时规则。
6. **开发计划安排文档**（涉及排期）：更新里程碑或任务。
7. **AGENTS.md**（涉及红线/约束）：更新对应章节。
8. **INDEX.md**（涉及文档结构）：同步更新文档路径和模块分布。
9. **decisions/**（涉及重大决策）：新增 ADR 文件记录决策，遵循格式：背景→方案对比→决策→后果。

### 第三步：记录决策
1. 更新 `docs/05-planning/task-status.json` 的 `decisions` 数组。
2. 记录决策时间、决策人和内容。

2. 提示用户："✅ 文档同步完成。"
