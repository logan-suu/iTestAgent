# ADR-002: 移除所有文档中的具体项目名引用

**状态**: 已接受
**日期**: 2026-07-12
**决策人**: AI Agent（文档净化任务）
**关联**: DOC-20260712-002

## 背景

原始文档多处将某具体 iOS 项目（`.xcworkspace`）表述为「目标项目」，包括：
- 项目概述中「当前目标项目形态」
- 各文档中作为示例引用

这与 iTestAgent 核心定位冲突：iTestAgent 是通用工具，不绑定任何特定项目。用户在任意 iOS 项目目录中启动 `itestagent` 即可工作。

## 决策

**通过 `git filter-branch` 从全部 Git 历史中移除项目名及 `.DS_Store` 文件。**

## 实施

1. 第一步（`--tree-filter`）：用 `sed` 从所有文件内容中删除项目名
2. 第二步（`--msg-filter`）：从提交信息中删除项目名
3. 第三步（`--index-filter`）：从 Git 索引中移除 `.DS_Store`
4. 新增 `.gitignore` 防止再次误提交

## 后果

### 正面
- 文档定位与产品定位一致（通用工具，不绑定项目）
- Git 历史干净，无泄露风险
- `.DS_Store` 防护到位

### 负面
- 少量示例失去了具体性（已用「示例项目」「无既有测试的 iOS 项目」替代）
