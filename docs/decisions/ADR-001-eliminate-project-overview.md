# ADR-001: 消除项目概述.md，内容分发至其他规格文档

**状态**: 已接受
**日期**: 2026-07-12
**决策人**: AI Agent（文档重组任务）
**关联**: DOC-20260712-001

## 背景

`docs/项目概述.md` 是项目最早创建的文档，包含产品定位、原则、GitHub Repo 复用矩阵、可行性分析等内容。随着其余 7 份规格文档逐步完善，项目概述与架构设计、技术选型、数据流、避坑手册等文档出现大量内容重复。维护两份重复内容会导致规格漂移。

## 决策

**消除项目概述.md，将其独有内容分发至对应规格文档。**

## 分发方案

| 独有内容 | 归属文档 |
|---|---|
| OpenCode 类比表 + 第一目标用户 | `AGENTS.md` §0 |
| GitHub Repo 详细推荐清单（最推荐/借鉴/不用） | `docs/02-architecture/技术选型文档.md` §12 |
| 用户使用流程示例 | 已在规格书 E4 覆盖 |
| 第一版明确不做清单 | `docs/01-spec/全量用户故事与验收标准规格书.md` 附录 |
| 可行性分析 + MVP 分档 + 去风险 MVP | `docs/03-implementation/开发避坑与关键注意点手册.md` §15 |
| Agent Session 模型示例 | `docs/02-architecture/架构设计文档.md` §8.0 |
| 必须支持的本地能力清单 | `docs/02-architecture/架构设计文档.md` §17 |

## 后果

### 正面
- 规格文档从 8 份精简为 7 份，消除重复维护负担
- 每份文档职责更清晰，查找效率提升
- 避免"改了 A 忘改 B"的规格漂移

### 负面
- 项目概述曾是一站式概览文档，删除后需要 README.md 承担对外展示职责
